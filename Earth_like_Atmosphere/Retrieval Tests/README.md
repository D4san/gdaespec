# Retrieval Tests

This directory contains the workflow for performing atmospheric retrieval tests on exoplanet spectra, specifically addressing the challenge of stellar contamination using **POSEIDON**.

## Strategies Compared

We compare three distinct retrieval strategies to assess the impact of stellar contamination:

1.  **No-contam ("Chem")**: Standard retrieval ignoring stellar contamination.
2.  **Contam ("Cont + Chem")**: Retrieval explicitly modeling stellar contamination.
3.  **G-DAE ("G-DAE + Chem")**: Our proposed method using a G-DAE to decontaminate observations before running a standard retrieval.

## Dependencies

*   **POSEIDON**
*   **MPI** (e.g., MPICH, OpenMPI)

## Workflow & File Structure

 The general workflow involves generating/processing data (if necessary), running the retrieval script, and then analyzing the results with the corresponding notebook. Use `plots.ipynb` for final paper figures.

### 1. Retrieval Scripts (`.py`)
These scripts perform the heavy-lifting of atmospheric retrieval. Run them on a cluster/multicore setup using MPI:
`mpirun -n <cores> python <script_name>.py`

*   **`Retrieval_no-contam_mpi.py`**: **No-contam Strategy**. Retrieves from contaminated spectra *without* a contamination model.
*   **`Retrieval_contam_mpi.py`**: **Contam Strategy**. Retrieves from contaminated spectra *using* a contamination model.
*   **`Retrieval_G-DAE_mpi.py`**: **G-DAE Strategy**. Retrieves from *decontaminated* spectra (generated by the G-DAE notebook).

### 2. Analysis Notebooks (`.ipynb`)
Each strategy has a dedicated notebook for setup, result analysis, and visualization.

*   **`Retrieval_analysis no-contam.ipynb`**: Analysis for the **No-contam** strategy.
*   **`Retrieval_analysis.ipynb`**: Analysis for the **Contam** strategy.
*   **`Retrieval_analysis G-DAE.ipynb`**: Analysis for the **G-DAE** strategy.
    *   **Important**: This notebook is also responsible for **generating the decontaminated observations** used by `Retrieval_G-DAE_mpi.py`. Run this *before* the G-DAE retrieval script.

### 3. Configuration & Manual Updates
The experiments iterate through specific ranges of stellar contamination. You must **manually replace** the contamination values in the scripts/notebooks to match the specific case you are testing:

*   **Stellar Contamination Ranges**:
    *   `0.00 - 0.000`
    *   `0.01 - 0.08`
    *   `0.08 - 0.54`
    *   `0.26 - 0.70`

Ensure parameters like `f_spot` and `f_fac` are updated accordingly in the files before running.
