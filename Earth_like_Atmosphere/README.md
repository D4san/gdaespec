# Experiment 0: G-DAE Performance on an Earth-like Atmosphere (TRAPPIST-1e Analog)

This directory contains the workflow used to train and evaluate the G-DAE on Earth-like transmission spectra for a TRAPPIST-1e analog.

## Data Source & Reference

The base spectral dataset used in this experiment is derived from [Duque-Castaño et al. (2025)](https://academic.oup.com/mnras/article/539/2/1528/8109635). The original source repository is:

- [MultiREx-public/examples/papers/DZF-MLBiosignatureClassification](https://github.com/D4san/MultiREx-public/tree/main/examples/papers/DZF-MLBiosignatureClassification)

The SPHINX stellar grids used to generate alternative contamination curves for
TRAPPIST-1 were downloaded separately and are documented inside
[stellar_contamination](stellar_contamination/).

## Core Workflow

Run the Earth-like experiment in this order:

1. `01_G-DAE.ipynb`
   Assemble the training set and train the G-DAE with the contamination curves
   stored in `stellar_contamination/`.
2. `02_G-DAE_Analysis.ipynb`
   Evaluate the trained G-DAE, compute reconstruction metrics, and analyze
   uncertainty estimates.
3. `Retrieval Tests/`
   Generate synthetic observations and run the five-observation POSEIDON
   campaign comparing G-DAE preprocessing against explicit
   stellar-contamination retrievals.

## Retrieval Campaign

The retrieval workflow lives in
[Retrieval Tests](Retrieval%20Tests/). It uses the clean PandExo spectrum,
the trained G-DAE model, and the PHOENIX/SPHINX contamination curves to build
five independent synthetic observations.

The campaign compares two strategies:

- G-DAE preprocessing followed by a chemistry-only POSEIDON retrieval.
- Joint atmospheric and stellar-contamination retrieval in POSEIDON.

The campaign record is stored in
[Retrieval Tests/campaign_5obs](Retrieval%20Tests/campaign_5obs/).

## Generated Products

The Earth-like workflow produces generated artifacts in addition to the source
data:

- `Models/G-DAE.keras`: trained G-DAE model used by the retrieval campaign.
- `plots/`: selected summary figures from the notebook analysis.
- `Retrieval Tests/campaign_5obs/*.csv`: compact retrieval-campaign summaries.

Large local outputs from full retrieval runs are kept inside ignored campaign
subfolders.
